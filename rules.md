# Project Rules & Permissions

## File Access
- **blueprint.md**: Read-only. This file contains the project requirements and must not be modified.
- **log.md**: Append-only. All actions must be recorded here with a timestamp. Existing entries must not be edited or deleted.

## Operational Constraints
- Follow the tech stack and modular architecture defined in `blueprint.md`.
- All code changes must be verified through testing or validation steps.
